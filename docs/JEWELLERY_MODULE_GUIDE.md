## 1. Overview <a name="overview"></a>

The Jewellery Module transforms BizArch ERP into a full-featured jewellery business management system. Unlike generic retail ERPs, it implements **proper jewellery accounting** with:

- **Metal Ledger** — Tracks fine gold weight (grams) alongside money for every transaction
- **Split COGS** — Separates metal cost from making charges in P&L
- **Karigar Management** — Gold issue/return tracking with GL integration
- **Status-Based Inventory** — Each item individually tracked (not FIFO quantities)
- **Gold Rate-Driven Pricing** — Prices calculated live from daily rates, never stored
- **Multi-Edition Compliance** — India (GST/HUID) and Saudi Arabia (VAT/SASO)

The module only activates when `isJewelleryModuleEnabled` is turned ON for an organization. Non-jewellery orgs are completely unaffected.

## 2. Setting Up a Jewellery Shop <a name="setting-up-a-jewellery-shop"></a>

### Step 1: Create Organization (Super Admin)
1. Login as super admin
2. Go to **Organizations** > **New Organization**
3. Set name, slug, and edition (India or Saudi Arabia)
4. Create a user for the organization

### Step 2: Enable Jewellery Module (Super Admin)
1. Click on the organization row to open settings
2. In the **Modules** section, toggle **Jewellery Module** ON
3. This automatically:
   - Creates 6 jewellery-specific GL accounts (1460, 1465, 4110, 4120, 5110, 5120)
   - Enables the Jewellery Shop sidebar section
   - Unlocks jewellery-specific settings

### Step 3: Configure Jewellery Settings
Go to the organization settings and configure:

| Setting | Default | Description |
|---------|---------|-------------|
| HUID Mandatory | Yes (India) | Require hallmark unique ID |
| SASO Mandatory | Yes (Saudi) | Require SASO certification |
| Gold Tax Rate | 3% (India) | GST on gold value |
| Making Charge Tax Rate | 5% (India) | GST on making charges |
| Default Wastage % | 5% | Default wastage for new items |
| Karigar Wastage Tolerance | 3% | Alert threshold for karigar wastage |
| Aging Alert Days | 180 | Alert for items in stock too long |
| Enabled Purities | K24, K22, K21, K18, K14, K9 | Which purities to show |
| Enabled Metals | Gold, Silver, Platinum | Which metals to track |

### Step 4: Configure Sidebar (Super Admin)
For a **jewellery-only shop**, hide irrelevant menu items:

**HIDE these items:**
- POS Terminal (unless using POS for jewellery counter sales)
- Products (jewellery items are managed in Jewellery Inventory)
- Quotations (unless doing custom order quotes)
- Purchase Invoices (unless tracking supplier purchases formally)
- Debit Notes
- Supplier Payments
- Expenses (unless tracking shop expenses)
- Journal Entries (auto-generated by the system)
- Chart of Accounts (system-managed)
- All Inventory section items (Branches, Stock Transfers, Opening Stock)
- Mobile Shop section (disable module)
- Restaurant section (disable module)

**KEEP these items:**
- Dashboard
- Customers
- Sales Invoices (for viewing generated invoices)
- Credit Notes (for returns)
- Customer Payments (for payment tracking)
- Cash & Bank
- **All Jewellery Shop items** (Dashboard, Gold Rates, Inventory, Old Gold, Karigars, Repairs, Schemes, Reports)
- Reports
- Settings

**Set default landing page** to: `/jewellery-shop/dashboard`

### Step 5: Set Today's Gold Rate
1. Go to **Jewellery Shop** > **Gold Rates**
2. Enter today's K24 (24 karat) buy and sell rates
3. The system auto-derives rates for K22, K21, K18, K14, K9

### Step 6: Create Jewellery Categories
Examples: Rings, Necklaces, Bangles, Earrings, Chains, Pendants, Bracelets

### Step 7: Add Karigars (Artisans)
Add your goldsmiths with their name, phone, specialization, and wastage tolerance.

### Step 8: Create Your First Customer
Add a customer with name, phone, and optionally GST details (for India).

## 3. Daily Operations Workflow <a name="daily-operations-workflow"></a>

### Morning Routine
1. **Set Today's Gold Rate** — Update K24 buy/sell rate. Other purities auto-derive.
2. **Check Dashboard** — Review stock count, fine weight, market value, pending repairs.
3. **Review Aging Alerts** — Items in stock >180 days need attention.

### Purchasing Gold / Receiving Items
1. **From Supplier (Purchase Invoice):**
   - Create Purchase Invoice with jewellery metadata (tag, purity, weight)
   - System auto-creates JewelleryItem record + metal ledger INFLOW
   - GL: DR Gold Inventory (1460) / CR Accounts Payable (2100)

2. **Manual Item Entry:**
   - Go to **Jewellery Inventory** > **Add Item**
   - Enter tag number, metal type, purity, weights, making charges, stone details
   - System calculates net weight and fine weight automatically

### Karigar (Artisan) Workflow
1. **Issue Gold** — Record gold given to karigar (weight + purity)
   - Metal ledger: OUTFLOW (gold leaves your stock)
   - GL: DR Gold with Karigars (1465) / CR Gold Inventory (1460)

2. **Receive Finished Piece** — Record returned items
   - Metal ledger: INFLOW (gold returns as finished piece)
   - GL: DR Gold Inventory (1460) / CR Gold with Karigars (1465)

3. **Record Wastage** — Normal gold loss during making
   - Metal ledger: OUTFLOW (gold permanently lost)
   - GL: DR Making COGS (5120) / CR Gold with Karigars (1465)

4. **Receive Scrap** — Leftover gold scraps returned
   - Metal ledger: INFLOW (scrap returns to your stock)
   - GL: DR Gold Inventory (1460) / CR Gold with Karigars (1465)

### Selling Jewellery
1. Go to **Jewellery Sale** or use the sale API
2. Select customer and items to sell
3. System automatically:
   - Calculates price from current gold rate + wastage + making charges + stones
   - Applies correct tax (India: 3% on gold + 5% on making; Saudi: 15% VAT)
   - Creates invoice (JINV-YYYYMMDD-XXX)
   - Marks items as SOLD
   - Creates split GL journals (Metal Revenue vs Making Revenue)
   - Records metal ledger OUTFLOW for each item
4. Optionally apply old gold credit to reduce the total

### Old Gold Exchange
1. Customer brings old gold
2. Test purity (XRF scanner — take 3 readings)
3. Record: weight, purity %, melting loss %, rate
4. System calculates total value and creates metal ledger INFLOW
5. Can be adjusted against a future invoice (one-time credit)

### Repairs
1. **Receive** — Log the item description, type, estimated cost
2. **In Progress** — Assign to karigar
3. **Ready** — Record actual cost, mark as ready for pickup
4. **Delivered** — Customer picks up, close the repair

### Customer Schemes
1. Create a savings plan: monthly amount, 11 months + 1 bonus month
2. Record monthly payments (cash/bank transfer/UPI)
3. After completion, customer can redeem against a jewellery purchase

## 4. Module Features <a name="module-features"></a>

### Jewellery Inventory
- **Individual item tracking** with unique tag numbers
- **RFID tag support** for automated scanning
- **Status management**: IN_STOCK, SOLD, IN_REPAIR, ON_APPROVAL, MELTED, IN_DISPLAY, CONSIGNMENT
- **Consignment tracking** with return deadlines
- **Stone details** with GIA/IGI certification
- **HUID** (India hallmark) and **SASO** (Saudi) compliance
- **Photo uploads** for each item
- **Bulk scan** for batch lookups

### Gold Rates
- Daily buy/sell rates per purity and metal type
- Auto-derive other purities from K24 base rate
- Rate locking for in-progress sales (prevents price changes mid-transaction)

### Pricing Engine
- **Pure function** — prices are NEVER stored on items, always calculated live
- Formula: `goldValue + wastageValue + makingCharges + stoneValue`
- Making charge types: per gram, percentage of gold value, or fixed amount
- Server-side and client-side mirrors for live UI updates

### Reports
| Report | What It Shows |
|--------|--------------|
| **Metal Balance** | Fine gold inflow vs outflow by purity, source type, and karigar |
| **Stock Valuation** | Current inventory valued at today's gold rate |
| **Inventory Aging** | Items in stock grouped by age (0-30, 31-90, 91-180, 180+ days) |
| **Karigar Reconciliation** | Gold issued vs returned per karigar, wastage analysis |
| **Gold Movement** | Gold flow in/out by source (purchases, sales, karigars, old gold) |

## 5. Metal Ledger (Fine Gold Accounting) <a name="metal-ledger"></a>

The metal ledger tracks every gram of fine gold entering or leaving your business. It's the jewellery equivalent of a financial ledger — but for weight.

### How It Works
Every transaction creates a `MetalLedgerEntry` with:
- **Direction**: INFLOW (gold coming in) or OUTFLOW (gold going out)
- **Fine Weight**: Pure gold equivalent (e.g., 10g of K22 = 9.167g fine gold)
- **Source Type**: PURCHASE, SALE, KARIGAR_ISSUE, KARIGAR_RETURN, OLD_GOLD_IN, WASTAGE, SCRAP, CREDIT_NOTE_RETURN, PURCHASE_RETURN

### Balance Rule
`Total Fine Gold in Stock = Sum(INFLOW) - Sum(OUTFLOW)`

This must match your physical stock's total fine weight. Any discrepancy indicates theft, unrecorded wastage, or data entry errors.

### What Creates Ledger Entries

| Event | Direction | Fine Weight Source |
|-------|-----------|-------------------|
| Purchase Invoice (jewellery) | INFLOW | Item's calculated fine weight |
| Jewellery Sale | OUTFLOW | Item's fine weight |
| Karigar Gold Issue | OUTFLOW | Issued weight x purity |
| Karigar Return | INFLOW | Returned weight x purity |
| Karigar Wastage | OUTFLOW | Wastage weight x purity |
| Karigar Scrap | INFLOW | Scrap weight x purity |
| Old Gold Purchase | INFLOW | Weight x purity% |
| Credit Note (return) | INFLOW | Item's fine weight |
| Debit Note (purchase return) | OUTFLOW | Item's fine weight |

## 6. GL Accounts & Financial Reports <a name="gl-accounts"></a>

### Jewellery-Specific GL Accounts

| Code | Account Name | Type | Purpose |
|------|-------------|------|---------|
| **1460** | Gold Inventory | Asset | Total value of gold in stock |
| **1465** | Gold with Karigars | Asset | Gold given to artisans (receivable) |
| **4110** | Jewellery Sales - Metal Value | Revenue | Gold component of sales |
| **4120** | Jewellery Sales - Making Charges | Revenue | Labor/craft revenue |
| **5110** | Jewellery COGS - Metal Cost | Expense | Gold cost of items sold |
| **5120** | Jewellery COGS - Making & Wastage | Expense | Labor cost + wastage |

### How Sales Appear in P&L

```
REVENUE
  4110 Metal Revenue:      60,000  (gold value of items sold)
  4120 Making Revenue:      9,562  (making charges + stones)
  ────────────────────────────────
  Total Revenue:           69,562

COST OF GOODS SOLD
  5110 Metal COGS:         45,000  (gold cost at purchase rate)
  5120 Making COGS:        10,000  (karigar wastage + making cost)
  ────────────────────────────────
  Total COGS:              55,000

GROSS PROFIT:              14,562
  Metal Margin:            15,000  (gold price appreciation)
  Making Margin:             -438  (making charges vs cost)
```

This split tells you whether your profit comes from gold price movement or from making charges — critical insight for a jewellery business.

### Karigar Journal Entries
| Transaction | Debit | Credit |
|------------|-------|--------|
| Gold Issue | 1465 Gold with Karigars | 1460 Gold Inventory |
| Gold Return | 1460 Gold Inventory | 1465 Gold with Karigars |
| Wastage | 5120 Making COGS | 1465 Gold with Karigars |
| Scrap Return | 1460 Gold Inventory | 1465 Gold with Karigars |

## 8. India Compliance <a name="india-compliance"></a>

### GST for Jewellery
- **Gold & Stones**: 3% GST (HSN 7113)
- **Making Charges**: 5% GST (SAC 9988)
- **Intra-state**: Split into CGST + SGST
- **Inter-state**: IGST (single rate)
- The system auto-detects inter-state based on org vs customer GST state codes

### HUID (Hallmark Unique Identification)
- Mandatory for all gold items sold in India (BIS regulation)
- Each item stores a unique HUID number
- Configurable via `jewelleryHuidMandatory` setting

### PAN Requirements
- PAN mandatory for purchases above threshold (default: Rs 2,00,000)
- Configurable threshold via `jewelleryPanThreshold`
- Applies to old gold purchases

### TCS (Tax Collection at Source)
- Optional TCS on jewellery sales above threshold
- Configurable rate and threshold

### Cash Transaction Limit
- Cash limit enforcement (default: Rs 2,00,000 per transaction)
- Configurable via `jewelleryCashLimitAmount`

## 9. Saudi Compliance <a name="saudi-compliance"></a>

### VAT for Jewellery
- **Regular jewellery**: 15% VAT
- **Investment gold (K24 pure gold)**: 0% VAT (zero-rated)
- Auto-detected based on purity

### SASO Certification
- SASO mark required for items sold in Saudi Arabia
- Fields: SASO mark, assayer code, year
- Configurable via `jewellerySasoMandatory`

## 12. Industry Standard Features We Don't Have Yet (Gap Analysis)

Based on research across 15+ dedicated jewellery ERPs (WinGold Next, Karat ERP, GehnaERP, PIRO, Marg, J-Soft, SEA ERP, ABHUSHAN), these are standard features we should consider adding:

### High Priority (Common in all jewellery ERPs)

| Feature | What It Is | Why It Matters |
|---------|-----------|----------------|
| **Girvi (Gold Loan/Pawn)** | Customers pledge gold jewellery as collateral for cash loans. Track KYC, interest calculations, loan maturity, locker allocation. | Standard revenue stream for Indian jewellers. Legally regulated. |
| **Approval/Memo (Home Trial)** | Customer takes items home on approval without payment. Track item, recipient, expected return date. Convert to sale or process return. | Very common in high-value jewellery sales. No other ERP module handles this. |
| **Barcode/RFID Tag Printing** | Print labels with tag number, weight, purity, HUID, barcode for each item. | Essential for retail operations — customers expect printed tags. |
| **Jewellery Invoice PDF** | Invoice showing weight breakdown, gold rate, making charges, stone details, HUID, old gold adjustment. | Current invoice template is generic. Jewellery customers expect weight details. |
| **Daily Gold Rate Auto-Fetch** | Pull live rates from IBJA (India Bullion Jewellers Association) or international feeds. | Saves 5 minutes daily, eliminates human error in rate entry. |
| **Counter/Tray Management** | Assign items to display counters/trays. Quick reconciliation via scanning. | Standard for shops with multiple display counters. |

### Medium Priority (Common in most jewellery ERPs)
